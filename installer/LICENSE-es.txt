Versión del acuerdo: 20 de julio de 2026

Lea atentamente este documento antes de instalar o utilizar el software.

instalación y el uso de Firulai Inventory Agent para Windows (en adelante, el
«Agente»), publicado por Redsauce para su utilización con la plataforma
Firulai.

Al marcar la casilla de aceptación y continuar con la instalación, usted
confirma que ha leído y acepta este Acuerdo, y que dispone de autorización

Agente.

1. FINALIDAD DEL AGENTE

El Agente es una herramienta de inventario técnico y apoyo a la gestión de
vulnerabilidades. Se instala como un servicio de Windows, recopila
periódicamente información técnica del equipo y la transmite a Firulai mediante
una conexión HTTPS para mantener el inventario y correlacionarlo con
vulnerabilidades conocidas.

El Agente no sustituye una auditoría de seguridad, un análisis profesional ni
las medidas de prevención, actualización, copia de seguridad o respuesta ante
incidentes que correspondan al usuario.

4. INFORMACIÓN RECOPILADA Y ENVIADA

Para prestar su función, el Agente puede recopilar y enviar a Firulai:

  - nombre del equipo (hostname), nombre de dominio completo (FQDN), UUID
    asignado, versión, edición, compilación y arquitectura de Windows, zona
    horaria, fecha de recopilación y versión del Agente;
  - información técnica de hardware, como modelo de CPU y datos identificativos
    y de modelo de los discos detectados;
  - nombres, versiones y origen del software y paquetes instalados, obtenidos
    del Registro de Windows y de gestores compatibles, como winget, Chocolatey,
    pip y npm;
  - versiones de componentes esenciales del sistema y estado técnico necesario
    para mantener el inventario; y
  - metadatos operativos, mensajes de error y registros necesarios para
    diagnosticar la ejecución y el envío.

El Agente utiliza el UUID para identificar el sistema y el token del Agente para
autorizar las comunicaciones con Firulai. El token se guarda localmente en un
archivo de configuración cuyos permisos se restringen a SYSTEM y
Administradores.

El Agente no está diseñado para recopilar deliberadamente contraseñas, claves
privadas ni el contenido de documentos del usuario. No obstante, nombres de
equipos, nombres de paquetes u otros metadatos técnicos podrían constituir
datos personales o información confidencial según el entorno. El usuario debe
evaluar esa circunstancia y aplicar las medidas y bases jurídicas necesarias.

5. FUNCIONAMIENTO EN SEGUNDO PLANO

Una vez instalado, el Agente:

  - se registra como servicio de Windows con inicio automático;
  - realiza un primer inventario tras la instalación;
  - intenta actualizar el inventario diariamente, normalmente a las 03:00, hora
    local, y puede recuperar una ejecución pendiente;
  - puede reintentar los envíos que no se completen correctamente; y
  - conserva localmente configuración, estado, inventario y registros técnicos.

El Agente necesita conectividad HTTPS con la infraestructura de Firulai. La
desinstalación elimina el servicio y los archivos locales del Agente, pero no
elimina automáticamente el inventario ya almacenado en Firulai. El sistema se
marca como desconectado o inactivo y los datos remotos deberán gestionarse
desde Firulai.
